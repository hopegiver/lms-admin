# LMS Admin 마법사(Wizard) 기능 가이드

## 개요

LMS Admin에서는 복잡한 다단계 설정을 쉽게 할 수 있도록 마법사(Wizard) 패턴을 제공합니다. 마법사는 초보 사용자도 단계별로 안내를 받으며 설정을 완료할 수 있도록 돕습니다.

---

## 현재 구현된 마법사

### 1. 강좌 생성 마법사 (Course Creation Wizard)

**경로**: `/learning/courses-wizard`
**접근**: 학습 관리 > 강좌 > "➕ 강좌 만들기" > "🪄 간편 생성"

#### 단계별 구성 (7단계)

1. **기본 정보**
   - 강좌명, 카테고리, 난이도
   - 강사, 설명, 썸네일

2. **커리큘럼 구성**
   - 섹션 및 차시 추가
   - 구조화된 학습 경로 설계

3. **콘텐츠 연결** (선택)
   - 차시별 콘텐츠 매핑
   - 동영상, 문서 등 학습 자료 연결

4. **평가 설정** (선택)
   - 시험 추가
   - 과제 추가

5. **수료 설정**
   - 수료증 템플릿 선택
   - 수료 조건 설정 (진도율, 점수)
   - 자동/수동 수료 처리 방식

6. **가격 설정** (선택)
   - 유료/무료 선택
   - 정가, 할인가
   - 판매 기간

7. **최종 검토**
   - 입력한 모든 정보 확인
   - 즉시 발행 또는 임시 저장

#### 특징

- **단계별 유효성 검사**: 필수 항목 미입력 시 다음 단계 진행 불가
- **건너뛰기 가능**: 3, 4, 6단계는 선택사항으로 건너뛰기 가능
- **임시 저장**: 언제든지 작성 중인 내용 저장 가능
- **진행률 표시**: 상단 프로그레스 바로 현재 진행 상황 확인
- **단계 이동**: 이전 단계로 돌아가거나 특정 단계로 바로 이동 가능

## 마법사가 필요한 추천 기능

### 우선순위 1: 상품 등록 마법사 ⭐⭐⭐⭐⭐

**경로**: `/commerce/products-wizard` (예정)
**필요 이유**:
- 이커머스 LMS에서 강좌 다음으로 중요한 기능
- 가격, 할인, 옵션, 배송, 재고 등 복잡한 설정
- 초보 관리자가 실수하기 쉬운 영역

**예상 단계**:
1. 기본 정보 (상품명, 카테고리, 설명)
2. 가격 설정 (정가, 할인가, 옵션별 가격)
3. 재고 관리 (재고 추적, 수량, SKU)
4. 배송 설정 (배송비, 배송 방법, 무게/크기)
5. SEO/메타데이터 (검색 최적화)
6. 최종 검토 및 발행

---

### 우선순위 2: 프로모션 생성 마법사 ⭐⭐⭐⭐⭐

**경로**: `/commerce/promotions-wizard` (예정)
**필요 이유**:
- 할인 조건 설정 실수는 매출에 직접 영향
- 조건부 로직이 복잡함 (최소 금액, 특정 상품, 사용자 그룹 등)
- 잘못된 설정으로 인한 금전적 손실 방지

**예상 단계**:
1. 프로모션 유형 선택 (쿠폰/이벤트/번들/포인트)
2. 할인 설정 (할인율/할인금액/무료배송)
3. 적용 대상 (특정 상품/카테고리/전체)
4. 조건 설정 (최소 구매금액, 사용자 그룹, 중복 사용)
5. 기간 및 수량 제한
6. 최종 확인 및 발행

---

### 우선순위 3: 조직 온보딩 마법사 ⭐⭐⭐⭐

**경로**: `/users/organizations-wizard` (예정)
**필요 이유**:
- B2B 기업 고객 등록 시 여러 설정 동시 필요
- 담당자, 구성원, 권한, 결제 정보 등 복잡
- 온보딩 과정 표준화로 고객 만족도 향상

**예상 단계**:
1. 조직 기본 정보 (회사명, 업종, 규모)
2. 담당자 계정 생성 (관리자 정보)
3. 구성원 일괄 등록 (CSV 업로드 또는 수동 입력)
4. 강좌 할당 및 권한 설정
5. 결제 정보 (청구서 발행, 결제 방법)
6. 계약 조건 확인

---

### 우선순위 4: 랜딩페이지 생성 마법사 ⭐⭐⭐

**경로**: `/site/pages-wizard` (예정)
**필요 이유**:
- 마케팅 담당자(비개발자)도 페이지 생성 가능
- 특정 목적별 템플릿 패턴 제공
- 빠른 마케팅 캠페인 실행

**예상 단계**:
1. 페이지 목적 선택 (강좌 소개/이벤트/회사 소개)
2. 템플릿 선택 (미리 디자인된 레이아웃)
3. 히어로 섹션 설정 (제목, 이미지, CTA)
4. 주요 섹션 구성 (기능, 후기, FAQ 등)
5. CTA/폼 설정 (행동 유도 요소)
6. SEO 설정 (메타 태그, OG 이미지)
7. 미리보기 및 발행

---

### 우선순위 5: 이메일 캠페인 마법사 ⭐⭐⭐

**경로**: `/marketing/email-campaigns-wizard` (예정)
**필요 이유**:
- 발송 대상, 콘텐츠, 일정 등 여러 단계 필요
- 잘못된 발송 방지 (테스트 발송 단계 포함)
- A/B 테스트 및 개인화 설정

**예상 단계**:
1. 캠페인 목표/유형 (뉴스레터/프로모션/공지)
2. 수신자 그룹 선택 (전체/특정 태그/구매 이력 기반)
3. 이메일 디자인 (템플릿 선택 및 커스터마이징)
4. 발송 일정 설정 (즉시/예약/반복)
5. 테스트 발송 (내부 테스트 이메일)
6. 최종 검토 및 발송

## 마법사 설계 원칙

### 1. 사용자 경험 (UX)

- **점진적 노출(Progressive Disclosure)**: 한 번에 하나의 단계만 보여주기
- **명확한 진행 상황**: 프로그레스 바와 단계 인디케이터
- **유연한 탐색**: 이전/다음/특정 단계 이동 가능
- **실시간 검증**: 각 단계별 입력값 즉시 검증
- **컨텍스트 도움말**: 각 필드에 설명 및 예시 제공

### 2. 기술 구현

- **상태 관리**: Vue 3 reactive data로 전체 마법사 상태 관리
- **유효성 검사**: computed 속성으로 단계별 진행 가능 여부 판단
- **모듈화**: 각 단계를 독립적인 컴포넌트처럼 구성
- **데이터 지속성**: 임시 저장 기능으로 작업 중단 후 재개 가능

### 3. 접근성

- **키보드 네비게이션**: Tab, Enter로 단계 이동
- **명확한 레이블**: 모든 입력 필드에 설명적인 레이블
- **에러 메시지**: 구체적이고 해결 방법을 제시하는 메시지
- **로딩 상태**: 비동기 작업 중 명확한 피드백

## 마법사 vs 직접 입력

### 마법사를 사용해야 할 때

- 처음 사용하는 기능
- 설정 순서가 중요한 경우
- 여러 연관된 설정을 한 번에 해야 할 때
- 실수 시 심각한 영향이 있을 때 (예: 가격 설정)

### 직접 입력을 사용해야 할 때

- 숙련된 사용자
- 특정 항목만 빠르게 수정할 때
- 고급 설정이 필요할 때
- 반복 작업 시 (API 연동 등)
